## 2) الهدف التنفيذي
تنفيذ متجر إلكتروني كامل لـ **سلة الحصاد** على Laravel على مرحلتين:

1. **مرحلة النقل الحرفي (بدون تعديل تصميمي):**
   - تحويل قالب الواجهة الأمامية (Fruitables) إلى مشروع Laravel بنفس التفاصيل الحالية.
   - بناء **لوحة تحكم Admin** كاملة.
   - إنشاء **الجداول + migrations** الأساسية.

2. **مرحلة التخصيص والتحسين لسلة الحصاد:**
   - مواءمة المحتوى والهوية والمنتجات مع نشاط سلة الحصاد.
   - إضافة **seeders** للبيانات الأولية.
   - تحسين تجربة الشراء والتوصيل وطرق الدفع بحسب سياسة المتجر.

## 3) نطاق العمل (Scope)

### A) واجهة المتجر (Storefront)
- Home
- Shop / Categories / Product Details
- Cart
- Checkout
- Contact
- About / FAQ / Terms / Privacy (إن وجدت)

### B) لوحة الإدارة (Admin Dashboard)
- إدارة المنتجات والأصناف
- إدارة المخزون
- إدارة الطلبات والحالات
- إدارة العملاء والعناوين
- إدارة كوبونات الخصم
- إدارة إعدادات المتجر (هوية، فروع، توصيل، دفع)
- إدارة السلايدر والبنرات والمحتوى
- التقارير الأساسية (مبيعات يومية/أسبوعية، أفضل المنتجات)

### C) البنية الخلفية (Backend)
- مصادقة وصلاحيات (Admin / Staff / Customer)
- REST endpoints أو Web Controllers حسب النمط
- خدمات الطلبات والدفع والشحن والإشعارات

## 4) التصميم التقني المقترح (Laravel)

### 4.1 تقسيم المشروع
- `resources/views/store/*` لواجهات المتجر.
- `resources/views/admin/*` لواجهات لوحة التحكم.
- `app/Http/Controllers/Store/*`
- `app/Http/Controllers/Admin/*`
- `app/Services/*` لخدمات الأعمال (CheckoutService, InventoryService...).

### 4.2 الجداول الأساسية (Migrations)

#### الهوية والمستخدمون
- `users`
- `roles`
- `role_user`
- `addresses`

#### الكتالوج
- `categories`
- `products`
- `product_images`
- `product_variants` (اختياري: وزن/حجم)
- `product_prices` (اختياري إذا يوجد تسعير مرن)
- `tags`
- `product_tag`

#### المخزون
- `inventories`
- `stock_movements`

#### السلة والطلبات
- `carts`
- `cart_items`
- `orders`
- `order_items`
- `order_status_logs`

#### الدفع والتوصيل
- `payment_methods`
- `payments`
- `shipping_zones`
- `shipping_rates`
- `shipments`

#### التسويق والمحتوى
- `coupons`
- `coupon_usages`
- `banners`
- `sliders`
- `pages`
- `settings`

#### التقييمات والتواصل
- `product_reviews`
- `contact_messages`

### 4.3 حالات الطلب المقترحة
`pending -> confirmed -> preparing -> out_for_delivery -> delivered`

وحالات بديلة:
`cancelled`, `failed`, `returned`.

## 5) خطة التنفيذ المرحلية (Implementation Roadmap)

## المرحلة 0: التحضير والتحليل (1-2 يوم)
- تحليل كامل لقالب Fruitables (HTML/CSS/JS/Assets).
- حصر الصفحات والمكونات المشتركة (header/footer/components).
- توثيق بيانات سلة الحصاد المطلوبة من السوشال/العميل.

**المخرجات:**
- Mapping document: (Template file -> Laravel view/component).
- قائمة حقول البيانات المطلوبة من العميل.

## المرحلة 1: النقل الحرفي للقالب إلى Laravel (2-4 أيام)
- نقل الأصول إلى `public/assets`.
- بناء Layout رئيسي Blade.
- تحويل كل صفحة HTML إلى Blade بنفس الشكل (بدون تغيير).
- إعداد Routing للصفحات الثابتة.

**المخرجات:**
- متجر يعمل على Laravel بنفس التصميم الحالي 1:1.

## المرحلة 2: لوحة الإدارة + المصادقة والصلاحيات (3-5 أيام)
- تركيب نظام auth.
- بناء RBAC (Admin / Staff).
- صفحات CRUD للإدارة الأساسية.

**المخرجات:**
- لوحة تحكم جاهزة لإدخال البيانات وإدارة المنتجات والطلبات.

## المرحلة 3: قاعدة البيانات والموديلات والمهاجرشنز (3-4 أيام)
- إنشاء الموديلات والعلاقات.
- كتابة migrations الأساسية + الفهارس.
- إعداد factories مبدئية للاختبار.

**المخرجات:**
- قاعدة بيانات سليمة قابلة للتوسع.

## المرحلة 4: ربط الواجهة بالداتا الحقيقية (3-5 أيام)
- عرض المنتجات/التصنيفات من DB.
- Cart و Checkout فعليين.
- إنشاء الطلبات وتغيير حالتها.

**المخرجات:**
- تدفق شراء كامل End-to-End.

## المرحلة 5: تخصيص سلة الحصاد + Seeders (2-4 أيام)
- إدخال الهوية البصرية والبيانات الفعلية.
- إنشاء seeders: تصنيفات، منتجات، بنرات، إعدادات التوصيل.
- تحسين نصوص واجهة المستخدم بالعربية/الإنجليزية.

**المخرجات:**
- نسخة مخصصة بالكامل لـ Harvest Basket Jo.

## المرحلة 6: الاختبارات والإطلاق (2-3 أيام)
- اختبارات وظيفية (Feature tests).
- اختبار checkout والحالات الحرجة.
- تحسين الأداء والكاش والـ SEO.

**المخرجات:**
- نسخة جاهزة للإطلاق (Production-ready baseline).
